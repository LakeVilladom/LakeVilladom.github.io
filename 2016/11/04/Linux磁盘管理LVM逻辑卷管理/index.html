  • LVM是 Logical Volume Manager（逻辑卷管理）的简写，它是Linux环境下对磁盘分区进行管理的一种机制。可以方便地将新的硬盘添加到LVM，以直接扩展已经存在的逻辑,LVM并不需重启就可以让内核知道分区的存在。

LVM结构图

概念及命令

分区(MBR、GPT)

  • MBR分区表：（MBR含义：主引导记录）
    　所支持的最大卷：2T （T; terabytes,1TB=1024GB）;
    　对分区的设限：最多4个主分区或3个主分区加一个扩展分区;
    　MBR硬盘分区有3种类型：主分区（Primary Partition）
    　　　　　　　　　　　　　扩展分区（Extended Partition）
    　　　　　　　　　　　　　逻辑分区（Logical Partition）
    　对分区有一些限制：
    　　　　　一个硬盘最多可以有四个主分区。
    　　　　　一个硬盘最多可以有一个扩展分区，扩展分区是个概念，看不到的，也没法直接使用。
    　　　　　一个扩展分区可以划分为多个逻辑分区。
    　分区工具：fdisk、parted
  • GPT分区表：（GPT含义：GUID分区表）
    　支持最大卷：18EB，（E：exabytes,1EB=1024TB）
    　每个磁盘最多支持128个分区（没有主分区、扩展分区的区别）
    　分区工具：gdisk、parted

LVM卷

  • 物理卷—–PV（Physical Volume）
    　物理卷在逻辑卷管理中处于最底层，它可以是实际物理硬盘上的分区，也可以是整个物理硬盘。
  • 卷组——–VG（Volumne Group）
    　卷组建立在物理卷之上，一个卷组中至少要包括一个物理卷，在卷组建立之后可动态添加物理卷到卷组中。一个逻辑卷管理系统工程中可以只有一个卷组，也可以拥有多个卷组。
  • 逻辑卷—–LV（Logical Volume）
    　逻辑卷建立在卷组之上，卷组中的未分配空间可以用于建立新的逻辑卷，逻辑卷建立后可以动态地扩展和缩小空间。系统中的多个逻辑卷要以属于同一个卷组，也可以属于不同的多个卷组。
  • 物理区块–PE（Physical Extent）
    　LVM 默认使用4MB的PE区块，物理区块是物理卷中可用于分配的最小存储单元，物理区域的大小可根据实际情况在建立物理卷时指定。物理区域大小一旦确定将不能更改，同一卷组中的所有物理卷的物理区域大小需要一致。

  • pvcreate
  • 要创建物理卷必须首先对硬盘进行分区，并且将硬盘分区的类型设置为”8e”后，才能使用pvcreate指令将分区初始化为物理卷。
  • 卷组直接用vgcreate扩展物理分区的时候也会先通过pvcreate初始化为物理
【语    法】pvcreate [选项] [参数]
【功能介绍】pvcreate指令用于将物理硬盘分区初始化为物理卷，以便被LVM使用。
【选项说明】
    -f   强制创建物理卷，不需要用户确认
    -u   指定设备的UUID
    -y   所有的问题都回答“yes”
    -Z   是否利用前4个扇区
【参数说明】
    物理卷    指定要创建的物理卷对应的设备文件名
  • vgcreate

　卷组（Volume Group）将多个物理卷组织成一个整体，屏蔽了底层物理卷细节。在卷组上创建逻辑卷时不用考虑具体的物理卷信息

【语    法】vgcreate [选项] [参数]
【功能介绍】vgcreate指令用于创建LVM卷组。
【选项说明】
    -l  卷组上允许创建的最大逻辑卷数
    -p  卷组中允许添加的最大物理卷数
    -s  卷组上的物理卷的PE大小
【参数说明】
    卷组名       要创建的卷组名称
    物理卷列表   要加入到卷组中的物理卷列表
  • lvcreate

　lvcreate命令用于创建LVM的逻辑卷。逻辑卷是创建在卷组之上的。逻辑卷对应的设备文件保存在卷组目录下，例如：在卷组”vg1”上创建一个逻辑卷”lvol0”，则此逻辑卷对应的设备文件为”/dev/vg1/lvol0”（软链接）,指向/dev/dm-*

【语    法】lvcreate [选项] [参数]
【功能介绍】lvcreate指令用于创建LVM的逻辑卷。
【选项说明】
    -L 指定逻辑卷的大小，单位为“kKmMgGtT”字节
    -l 指定逻辑卷的大小（LE数）
    -n 后面跟逻辑卷名
  • lvextend

　使用lvextend指令动态在线扩展磁盘空间，整个空间扩展过程对于应用程序来说是完全透明的。调整逻辑卷大小的功能是LVM最有用的功能。首先扩展物理边界，然后扩展文件系统边界。

[lkvm@Cloud~]$ sudo xfs_growfs /dev/vg1/lvol0       #使用xfs_growfs扩展xfs格式的文件系统
[lkvm@Cloud~]$ sudo resize2fs /dev/vg1/lvol0 16M    #使用resize2fs扩展eext3、ext4格式的文件系统

【语    法lvextend [选项] [参数]
【功能介绍】lvextend指令用于在线扩展逻辑卷的空间大小，而不中断应用程序对逻辑卷的访问。
【选项说明】
    -L指定逻辑卷的大小，单位为“kKmMgGtT”字节
    -l指定逻辑卷的大小（LE数）
【参数说明】
    逻辑卷     指定要扩展空间的逻辑卷
  • lvreduce

　缩减LVM卷大小,命令的顺序相当重要。注意：缩减危险系统很高，操作请谨慎。减少逻辑卷的空间大小一定要大于储存的数据大小，否则存储在后面的数据会丢失或者损坏

【语    法】lvreduce [选项] [参数]
【功能介绍】lvreduce指令用于减少LVM逻辑卷占用的空间大小。
【选项说明】
    -L  指定逻辑卷的大小，单位为“kKmMgGtT”字节
    -l  指定逻辑卷的大小（LE数）
【参数说明】
    逻辑卷   指定要操作的逻辑卷对应的设备文件
  • mkfs

　分区后格式化是没有用的，最终你在创建lv的时候还需要再格式化一次，否则不能直接用，必须在创建逻辑卷以后格式化

【语    法】mkfs [-V] [-t fstype] [fs-options] filesys [blocks]
【功能介绍】Linux mkfs命令用于在特定的分区上建立 linux 文件系统
【选项说明】
    device ： 预备检查的硬盘分区，例如：/dev/sda1
    -V : 详细显示模式
    -t : 给定档案系统的型式，Linux 的预设值为 ext2
    -c : 在制做档案系统前，检查该partition 是否有坏轨
    -l bad_blocks_file : 将有坏轨的block资料加到 bad_blocks_file 里面
    block : 给定 block 的大小
  • mount
  • /etc/mtab用户不可以修改​,记录的是当前系统挂载的分区的信息，记录系统当前已经挂载的文件系统，随着系统的运行在不断变化。
  • /etc/fstab 用户可以修改更像配置文件，系统在启动时加载读取/etc/fstab文件内容，自动挂载相应的文件系统。
  • mount命令用于加载文件系统到指定的加载点。Linux并不会自动挂载，必须使用Linux mount命令来手动完成挂载。
【语    法】 mount [选项][参数]
【选项说明】
    -V：显示程序版本； -l：显示已加载的文件系统列表； -h：显示帮助信息并退出
    -v：冗长模式，输出指令执行的详细信息； -n：加载没有写入文件“/etc/mtab”中的文件系
    -r：将文件系统加载为只读模式； -a：加载文件“/etc/fstab”中描述的所有文件系统。
【参数说明】
    设备文件名：指定要加载的文件系统对应的设备名；
    加载点：指定加载点目录。

  • snapshot:快照组

    　快照卷是将元数据重新复制一份做为快照卷访问入口，当数据本身发生变化时，将数据复制一份到快照空间，然后修改原数据，快照卷主要用于数据备份，创建的快照卷必须要和逻辑卷在同一卷组上，并且快照卷大小要大于等于快照卷存在周期内数据增长量。

创建快照使用命令：lvcreate
  lvcreate -L #[mMgGtT] -p r -s -n snapshot_lv_name(快照卷名字) original_lv_name(原卷名字)
    -L：指定快照卷大小
    -p: 指权限，一般为只读
    -s: 指创建为快照卷
    -n：指定快照卷名字
如：# lvcreate -L 6G -p r -s -n snap_mysql /dev/volume-group1/mysqldata
    删除快照使用命令：lvremove
  lvremove  snapshot_lv_name

LVM实战

　gdisk等工具磁盘分区—>创建物理卷（PV）—>创建卷组（Vg）—>创建逻辑卷（LV）—>格式化逻辑卷（mkfs)—>挂载分区(mount)—>调整逻辑卷（LV）—>调整卷组（Vg）—>调整物理卷（PV)—>删除磁盘分区

【注意:】对于物理卷、卷组以及逻辑卷的创建有严格顺序的，同样，对于逻辑卷、卷组以及物理卷的删除也是有严格顺序要求的，随意移除已加入卷组的磁盘可能会导致系统无法启动。

  1. 首先通过umount命令卸载正在使用的逻辑卷;
  2. 然后通过lvremove命令删除逻辑卷;
  3. 接下来通过vgremove命令删除卷组;
  4. 最后通过pvremove命令再来删除我们的物理卷。
