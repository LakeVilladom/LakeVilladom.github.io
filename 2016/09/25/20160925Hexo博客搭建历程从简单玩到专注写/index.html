  • 缘起:记录下与Hexo从初识到到相知的点滴！

简单玩：

  1. 初识Hexo，应该是从博客右下角的 “Powered by hexo and Theme by Yelee ” 。

  2. 2016年9月9日下午4点购买域名ruicloud.me(29元/年，第二年为什么价格翻几番啊！)，晚上11点购买一月的腾讯云主机（65元/月，其中使用了30元的代金券）。

  3. 2016年9月10日深夜2点成功搭建个人Hexo博客，杵下了第一条博客，并通过域名访问。

  4. 接下来的几天就是对博客的美化设置了，如头像、代码高亮、不算子统计、评论以及Hexo Admin插件以及Nginx反向代理以及直接在Nginx下部署等，期间也简单写了堆砌了几篇博客。

  5. 一个月的云主机很快会到期的,就有一个很迫切的问题，就是备份问题，尤其是.md文件的备份。接下来准备购买阿里云主机（选720/年的基础型的，包含了2个5G的云硬盘），单独购买2个硬盘主要是考虑想将/home单独到一个硬盘上(ubuntu上upgrade了一下，服务器差点重启不起来了，惊出了一身冷汗，现在重启一次要近20分钟)。

  6. 突然想到云主机安装过程中磁盘的分配不是用户可以控制的，怎么能将/home指定硬盘呢，就想在虚拟机下安装个Linux，试验一下。

  7. 这一试验就折腾出了一套自己的备份方案，决定不购买云主机了，毕竟下来小一千块钱呢…..

专注写：

因为hexo的搭建过程本省比较简单，接着最多也就是换个喜欢的主题，完善下配置，插件等。玩的阶段也就结束了，接下来，返璞归真，终归要过度到专注于写的层级上，这就得综合考虑安全、便捷以及性价比等因素了，综合以上因素最终制定出自己的hexo搭建及配置方案：

1、虚拟机安装CentOS 7，/home分区单独到一个虚拟磁盘上，保证源文件的独立。

　　当然在win上直接搭建hexo更方便，但是bige略low，玩嘛。
　　使用俩虚拟硬盘，直接将/home单独装到其中一个硬盘上，像我这种安装玩Linux先备份虚拟硬盘，一言不合就删了虚拟硬盘重新玩的人来说，将/home单独到一个磁盘上真的很重要。　

2、博客源文件及主题文件等用hexo-git-backup备份到国内coding、oschina上(速度块、能创建私有仓库)。

　　主要是github免费用户不能创建私有仓库，速度也慢，而coding速度也快，中文界面，现在应该也是比较稳定。

3、博客静态文件通过hexo-deployer-git托管到github和coding上(主要也是coding，速度快)。

　　域名还是主要指定在coding上，无他，速度快。

虚拟机安装CentOS 7

主要考虑要能让/home磁盘挂载到其他服务器上即能被识别使用！

设备类型 卷组 硬盘 挂载点 文件系统
1 标准分区 　　 无 　　 sda /boot xfs
2 标准分区 无 sdb /home xfs
3 LVM vghome sdb /home xfs
4 LVM vgcentos sda 无 swap
5 LVM vgcentos sda / xfs

其中2、3是二选一的，当选择了不同的设备类型时，在其他服务器上的挂载方式有所区别：

  • 选择”标准分区”时,直接挂载/dev下sdb即可:
[lkvm@LKVM-Cloud]$ sudo mount /dev/sdb /mnt
  • 选择”LVM”时，会在/dev下有卷组名相同的文件夹，挂载文件夹下文件方可
[lkvm@LKVM-Cloud]$ ls-l /dev/vghome/home   
  lrwxrwxrwx.  1 root root  7 9月  26 18:23 home -> ../dm-2
[lkvm@LKVM-Cloud]$ sudo mount /dev/vghome/home  /mnt  #成功
[lkvm@LKVM-Cloud]$ sudo mount sdb /mnt/  #这样挂载报错
  mount: you must specify the filesystem type
[lkvm@LKVM-Cloud]$ sudo mount sdb1 /mnt/  #这样挂载也报错
  mount: unknown filesystem type 'LVM2_member'

hexo的搭建(可参考hexo官网)

  1. 安装配置git

    [lkvm@LKVM-Cloud]$ sudo yum install git
[lkvm@LKVM-Cloud]$ git config --global user.name "Account"          # Github 用户名Account
[lkvm@LKVM-Cloud]$ git config --global user.email "mail@gmail.com"  # Github 邮箱mail@gmail.com

  2. 配置ssh密钥：直接在~/.ssh下放置公钥rsa_nkey.lv并设定600权限，再配置/etc/ssh/ssh_config：

    [lkvm@LKVM-Cloud]$ chmod 600 ~/.ssh/rsa_nkey.lv
[lkvm@LKVM-Cloud]$ vim /etc/ssh/ssh_config
IdentityFile ~/.ssh/rsa_nkey.lvc   #添加

  3. 如果rsa_nkey.lv权限不是600，会报如下错误：

    Permissions 0664 for '/home/lkvm/.ssh/rsa_nkey.lvc' are too open.
It is required that your private key files are NOT accessible by  others.

  4. 同时在github、coding下添加对应的私钥，然后测试：

    [lkvm@LKVM-Cloud]$ ssh -T git@github.com
Hi ! You've successfully authenticated, but GitHub does not provide shell access.

  5. 安装hexo

    [lkvm@LKVM-Cloud]$ curl --silent --location https://rpm.nodesource.com/setup_6.x | bash -  #安装nvm
[lkvm@LKVM-Cloud]$ yum -y install nodejs   #安装nodejs  
[lkvm@LKVM-Cloud]$ npm install -g hexo-cli
[lkvm@LKVM-Cloud]$ hexo init <folder>
[lkvm@LKVM-Cloud]$ cd <folder>
[lkvm@LKVM-Cloud]$ npm install
[lkvm@LKVM-Cloud]$ vim /etc/bashrc  #hexo命令必须在hexo安装目录内执行
alias hexo="cd ~/hexo&&hexo"

    更详细的nodejs安装过程可参考:Node.js安装配置

  6. 设置hexo server开机启动

    　　CentOS7使用systemd做为系统进程启动管理工具来替换了原有的SysV,现阶段很多Linux发行版也都拥抱了systend，正好借这个机会简单实践下通过systemd管理服务

    § 实践中 §

  7. 安装并配置主题yelee

    [lkvm@LKVM-Cloud]$ git clone https://github.com/MOxFIVE/hexo-theme-yelee.git themes/yelee    #配置参考文档


    删除yelee主题下的.git文件夹，否在在用hexo-git-backup备份该主题的时候备份空文件夹

Hexo配置实现备份

  1. 安装并配置hexo-deployer-git，直接将静态文件发布到git仓库

    [lkvm@LKVM-Cloud]$ npm install hexo-deployer-git --save  #安装
[lkvm@LKVM-Cloud]$ vim _config.yml   #配置_config.yml
deploy:
 type: git
 repo:
   coding: git@git.coding.net:XXX/xxx.git,coding-pages
   github: git@github.com:XXX/xxx.github.io.git,master

  2. 安装在线可视化写博客写作插件hexo-admin

    [lkvm@LKVM-Cloud]$ npm install --save hexo-admin

  3. 博客源码备份(使用hexo-git-backup插件)：

    [lkvm@LKVM-Cloud]$ npm install hexo-git-backup --save
[lkvm@LKVM-Cloud]$ git remote add coding git@git.coding.net:XXX/xxx.git
[lkvm@LKVM-Cloud]$ git remote -v
coding    git@git.coding.net:XXX/xxx.git (fetch)
coding    git@git.coding.net:XXX/xxx.git (push)
[lkvm@LKVM-Cloud]$ vim _config.yml   #配置_config.yml
backup:
  type: git
  repository:
    coding: git@git.coding.net:XXX/xxx.git,master
    oschina: git@git.oschina.net:XXX/xxx.git,source

    hexo-git-backup配置过程中一直报错： 

    fatal: 'github.com' does not appear to be a git repository
fatal: Could not read from remote repository.
Please make sure you have the correct access rights
and the repository exists.

    折腾了很久，后来如下添加远程git地址才成功。 

    [lkvm@LKVM-Cloud]$ git remote add coding git@git.coding.net:XXX/xxx.git

    不懂原理就只得多花费时间多折腾啊！

Dockerfile制作hexo博客的docker镜像 (ADD 20161102)

1、将hexo搭建过程制作成docker镜像，抽象为专注于写的博客工具。

2、作为第一个Dockerfile的练习项目。

  • 磁盘分区
设备类型 卷组 硬盘 挂载点 文件系统
1 标准分区 　　 无 　　 sda /boot xfs
2 LVM vghexo sdb /hexo xfs
3 LVM vghome sdc /home xfs
4 LVM vgcentos sda 无 swap
5 LVM vgcentos sda / xfs

创建/hexo分区并单独放到磁盘sdc上，用于放Hexo源文件，然后挂载到容器中

  • 方案一：
    刚开始尝试如上面搭建hexo博客一样搭建完整的hexo博客，然后通过脚本，完成github上托管源文件的拉取等初始化工作，优点是生成的docker镜像即是一个完整干净的hexo镜像，缺点该过程编写初始化脚本过程复杂不易控制，原有的插件以及配置都要重新配置，通过shell修改yaml难度大。
  • 方案二：
    考虑将托管在github上的博客源内容拉取到宿主机/hexo下，然后在docker run时通过-v挂载至容器中，该方案更加灵活，也符合Docker对于动态数据的处理，缺点是/node_modules目录也要通过github托管，在博客运行、插件升级时可能会出问题，而且上传、拉取都会费事。（现阶段使用该方案尚未发现验证严重问题）
  • Dockerfile编写
FROM alpine
MAINTAINER LakeVilladom, lakevilladom@aliyun.com
RUN \
    echo "http://mirrors.ustc.edu.cn/alpine/v3.4/main">/etc/apk/repositories&& \
    echo "http://mirrors.ustc.edu.cn/alpine/v3.4/community">>/etc/apk/repositories&& \
    apk --update --no-progress add git openssh nodejs&& \
    cd /root/&& \
    touch .npmrc&& \
    echo "registry = https://registry.npm.taobao.org">.npmrc && \
    npm install -g hexo-cli
    #hexo init hexo && \
    #cd hexo && \
    #npm install && \
    #npm install hexo-deployer-git --save && \
    #npm install hexo-git-backup --save && \
    #npm install --save hexo-tag-aplayer
VOLUME ["/root/.ssh","/root/hexo"]
EXPOSE 4000
ENTRYPOINT ["/bin/sh"]
WORKDIR /root/hexo

1、镜像基于alpine，体积更小,只有不到5M,基于上面Dockerfile制作的docker镜像大小为67.87M，Alpine Linux详细使用可参考Alpine Linux配置使用

2、打开注释部分可制作完整干净的hexo镜像，也可基于个人定制修改。

  • 拉取hexo源文件至/hexo/XXX/
git clone git.coding.net:XXX/XXX.git
  • build Docker镜像
docker build -t ihexo .
  • run Docker镜像
docker run -v /home/XXX/.ssh/XXX.lvc:/root/.ssh/id_rsa  -v /hexo/XXX/:/root/hexo/ -p 80:4000 --name chexo -it ihexo
  • 设置Docker开机启动
 systemctl enable docker
  • 将用户添加到Docker用户组
 sudo gpasswd -a ${USER} docker
 或
 usermod -aG docker your_username
  • 启动chexo
docker start chexo
docker attach chexo
  • 发布博客内容
[lkvm@LKVM-Cloud]$  vim hexo-admin.sh
  #!/usr/bin/env sh
  hexo clean&&hexo b&&hexo g&&hexo d
[lkvm@LKVM-Cloud]$ sh hexo-admin.sh

　　简单通过Dockerfile制作的基于hexo博客的docker镜像就完成了，折腾过程中最大的感受就是：慢！慢！慢！墙！墙！墙！社会主义真是急死人！

域名绑定

  • 域名解析

    参考网上教程，默认、海外分别指向coding和github,可通过路由跟踪工具跟踪域名访问情况。

  • github上的域名绑定

在source下新建CNAME(大写，无后缀)，在里面填写自己的域名，保存。

/hexo/source # cat CNAME
      www.lakevilladom.com
      lakevilladom.com
  • coding上的域名绑定

此处应该和github添加CNAME异曲同工，只是更加方便、人性化。
