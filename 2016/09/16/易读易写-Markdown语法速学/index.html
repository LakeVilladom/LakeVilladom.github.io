一、概述

  Markdown是一个Web上使用的文本到HTML的转换工具，可以通过易读易写的文本格式生成结构化的HTML文档。

二、基本语法

标题

  • setext形式：
       标题1
===================
       标题2
-------------------
  • atx形式：
# 一级标题 ##
## 二级标题 ##
### 三级标题  ##
#### 四级标题 ##
##### 五级标题 ##
###### 六级标题 ##

效果:

似乎没法演示

说明:

1、Setext 形式中= 和 - 的数量是没有限制的。通常的做法是使其和标题文本的长度相同，这样看起来比较舒服。或者可以像我一样，用四个 - 或 =。
2、Setext 形式只支持 h1 和 h2 两种标题，所以更常使用atx形式。
3、atx形式下的后面的##可有可无，可多可少，做闭合用。

段落与换行

1.单一段落< p > 用一个空白行;

2.连续两个空格 会变成一个< br >;

3.连续3个符号，然后是空行，表示 hr横线;

4.不分段换行：行末加两个空格，直接回车;

5.分段换行：两个或多个回车（用空行来换行）。

转义字符

`` \\ 反斜杠 \ 反引号
* 星号
_ 下划线
{} 大括号
[] 中括号
() 小括号
# 井号
+ 加号
- 减号
. 英文句号
! 感叹号

三、扩展语法(不完全通用)

内容目录[TOC]

  • 使用 [TOC] 以显示目录结构

    在段落中填写 [TOC](Table of contents) 以显示全文内容的目录结构。

  • 生成文章目录的插件: HEXO-TOC
[LV@LV-Cloud]$ npm install hexo-toc --save  #安装hexo-toc
[LV@LV-Cloud]$ vim ~/Hexo/_config.yml       #添加如下内容
toc:
  maxdepth: 3
  class: toc
  slugify: transliteration
  anchor:
    position: after
    symbol: '#'
    style: header-anchor

maxdepth :表示目录深度为3，即最多生成三级目录；
class :toc的CSS类，默认为false；

  • 使用HEXO支持的 TOC() 函数
    语法：
    解析内容中的标题标签 (h1~h6) 并插入目录。
    <%- toc(str, [options]) %>
参数 描述 默认值
class Class 名称 toc
list_number 显示编号 true
  • 不需要安装额外插件！
  • 不需要在文章中插入标记！
  • 不需要在配置文件添加配置！
  • 你的文章页模版中，插入调用辅助函数的代码!
    <%- toc(page.content, {
      class: 'post-toc',  
      list_number: true
  }) %>

说明：
  有一些编辑器不支持[TOC]标记，比如简书。

注脚footnotes

Hexo默认的markdown渲染插件好像不支持脚注，只能通过安装第三方插件实现。

  • 安装hexo-renderer-markdown-it插件
sudo npm un hexo-renderer-marked --save      #卸载系统默认插件hexo-renderer-marked
sudo npm i hexo-renderer-markdown-it --save   #安装 hexo-renderer-markdown-it插件
cd node\_modules/hexo-renderer-markdown-it/   #插件文件下
npm install markdown-it-footnote --save      #添加脚注
npm install markdown-it-emoji --save        #添加 Eomji 
npm install markdown-it-sub --save         #添加下标
npm install markdown-it-sup --save         #添加下标
  • 博客配置_config.yml文件
markdown:
  render:
  html: true # 兼容 HTML 语法 
             # 否则 <!--more--> 语法会失效
  plugins:
      - markdown-it-footnote
      - markdown-it-sup
      - markdown-it-sub
      - markdown-it-emoji
