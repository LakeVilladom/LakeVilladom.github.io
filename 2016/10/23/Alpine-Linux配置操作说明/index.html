Alpine Linux配置使用

By Lake Villadom

发表于 2016-10-23

文章目录

  • 缘起：最近在构建Docker镜像时使用Alpine Linux作为基础镜像,故有必要学习Alpine Linux，在此记录学习成果。

概述

Alpine Linux的Docker镜像特点是轻巧（大小只有5M）且有完整的包管理工具（APK），且以后Docker官方镜像可能会用Alpine Linux替换Ubuntu 。

系统配置（待补充）

  • docker镜像源修改

设置Daocloud加速器,因为Daocloud官方脚本不支持Alpine自动设置,需要自己修改/etc/conf.d/docker，并重启docker服务。

curl -sSL https://get.daocloud.io/daotools/set_mirror.sh | sh -s http://xx.m.daocloud.io  #不支持Alpine设置
sudo echo "DOCKER_OPTS=\"$DOCKER_OPTS --registry-mirror=http://xxx.m.daocloud.io\"" >> /etc/conf.d/docker
  • Alpine的Docker官方镜像中服务管理

Alpine的Docker官方镜像中没找到service管理工具，查看官网需要单独安装OpenRC:

Alpine Linux uses OpenRC for its init system.

安装完成后可以使用service，但是有报错。

网络配置

  • hostname配置
lkvm-alpine:~# echo "hostname" >/etc/hostname  
lkvm-alpine:~# hostname -F /etc/hostname      #使配置生效
  • DNS配置
lkvm-alpine:~# vim /etc/resolv.conf
nameserver 8.8.8.8
nameserver 114.114.114.114
  • ip配置
lkvm-alpine:~# /etc/network/interfaces
 #lo
    auto lo
    iface lo inet loopback
#dhcp自动获取配置ip
    auto eth0
    iface eth0 inet dhcp
#static配置静态ip
    iface eth0 inet static
          address 192.168.1.169
          netmask 255.255.255.0
          gateway 192.168.1.1
  • iptables防火墙安装
apk add iptables
apk add ip6tables
apk add iptables-doc
  • network服务重启
lkvm-alpine:~# /etc/init.d/networking restart或service networking restart

apk包管理

  • 镜像源配置

官方镜像列表：http://rsync.alpinelinux.org/alpine/MIRRORS.txt

MIRRORS.txt中是当前Alpine官方提供的镜像源（Alpine安装的时候系统自动选择最佳镜像源），但是列表中并未列出国内几个包含国内较好镜像源，列举如下：

清华TUNA 镜像源：https://mirror.tuna.tsinghua.edu.cn/alpine/
中科大镜像源：http://mirrors.ustc.edu.cn/alpine/
阿里云镜像源：http://mirrors.aliyun.com/alpine/

　镜像源的配置文件在/etc/apk/repositories下,这里配置apk镜像源为中科大镜像源，速度不错，安装包也较完整，包含了git、docker等，（相比较阿里云就没有docker） 　

lkvm-alpine:~# vi /etc/apk/repositories
#/media/cdrom/apks
http://mirrors.ustc.edu.cn/alpine/v3.4/main
http://mirrors.ustc.edu.cn/alpine/v3.4/community
  • apk包管理命令

通过apk --help命令查看完整的包管理命令，下面列举常用命令：

  1. update:从远程镜像源中更新本地镜像源索引

    update命令会从各个镜像源列表下载APKINDEX.tar.gz并存储到本地缓存，一般在/var/cache/apk/(Alpine在该目录下)、 /var/lib/apk/ 、/etc/apk/cache/下。

    apk update

  2. add:安装PACKAGES并自动解决依赖关系

    add命令从仓库中安装最新软件包，并自动安装必须的依赖包,也可以从第三方仓库添加软件包。

    apk add openssh openntp vim
apk add docker --update-cache --repository http://mirrors.ustc.edu.cn/alpine/v3.4/main/ --allow-untrusted

  3. del:卸载并删除PACKAGES

    apk del openssh openntp vim

  4. upgrade:升级当前已安装的软件包（一般包括内核）

    upgrade命令升级系统已安装的所以软件包（一般包括内核），当然也可指定仅升级部分软件包（通过-u或--upgrade选择指定）。

    apk update    #更新最新本地镜像源
apk upgrade   #升级软件
apk add --upgrade busybox   #指定升级部分软件包

  5. search:搜索软件包

    search命令搜索可用软件包，-v参数输出描述内容，支出通配符，-d或--description参数指定通过软件包描述查询。

    apk search            #查找所以可用软件包
apk search -v         #查找所以可用软件包及其描述内容
apk search -v 'acf*'  #通过软件包名称查找软件包
apk search -d 'docker'   #通过描述文件查找特定的软件包

  6. info:列出PACKAGES或镜像源的详细信息

    info命令用于显示软件包的信息，，-a参数显示详细信息。

    apk info              #列出所有已安装的软件包
apk info -a git       #显示完整的软件包信息
apk search -v         #查找所以可用软件包及其描述内容
apk search -v 'acf*'  #通过软件包名称查找软件包
apk search -d 'docker'   #通过描述文件查找特定的软件包
